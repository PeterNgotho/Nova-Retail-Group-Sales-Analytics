# Data

This folder contains the full Nova Retail Group dataset as five CSV files.

## Files

| File                | Rows    | Size  | Description                                       |
| ------------------- | ------- | ----- | ------------------------------------------------- |
| `dim_date.csv`      | 1,096   | ~65KB | Calendar table, 2022-01-01 to 2024-12-31          |
| `dim_product.csv`   | 20      | ~2KB  | Products across 4 categories                       |
| `dim_store.csv`     | 10      | ~1KB  | Stores across 5 South African provinces           |
| `dim_customer.csv`  | 5,000   | ~385KB| Customer master with demographics and loyalty    |
| `fact_sales.csv`    | 50,000+ | ~2.4MB| Sales transactions at line-item grain             |

## About the dataset

The dataset is **fully synthetic** — no real customer, sales, or store data is included. It was generated programmatically with realistic characteristics:

- 3 years of sales history (2022 → 2024) with realistic year-on-year growth
- Strong December seasonality and softer January/February periods
- Gauteng and Western Cape over-indexed for revenue (mirroring real South African retail patterns)
- Electronics is the dominant category by revenue
- Loyalty tier distribution skews Bronze with a small Platinum elite
- Mixed payment methods weighted toward Card

## Data quality

The dataset was deliberately seeded with realistic data quality issues — orphan keys, duplicates, range violations, missing emails, and casing inconsistencies. See [`../docs/data-quality.md`](../docs/data-quality.md) for the full audit and resolution decisions.

This is intentional. A clean dataset would teach you nothing about working with real retail data.

## Loading the CSVs into PostgreSQL

After running [`../sql/01_create_schema.sql`](../sql/01_create_schema.sql) to create the tables:

```sql
\COPY dim_date     FROM 'data/dim_date.csv'     WITH (FORMAT CSV, HEADER TRUE);
\COPY dim_product  FROM 'data/dim_product.csv'  WITH (FORMAT CSV, HEADER TRUE);
\COPY dim_store    FROM 'data/dim_store.csv'    WITH (FORMAT CSV, HEADER TRUE);
\COPY dim_customer FROM 'data/dim_customer.csv' WITH (FORMAT CSV, HEADER TRUE);
\COPY fact_sales   FROM 'data/fact_sales.csv'   WITH (FORMAT CSV, HEADER TRUE);
```

Note: `\COPY` may fail if the data quality issues violate constraints (out-of-range discounts, orphan customer keys). Drop the foreign key and check constraints before loading, or load into staging tables first and clean before merging.

## Loading the CSVs into Power BI directly

Skip the database entirely if you'd prefer:

1. Open Power BI Desktop
2. **Get data** → **Text/CSV**
3. Load each of the five CSV files
4. Set up relationships matching the [ERD](../images/erd_diagram.png)
5. Mark `dim_date` as the official Date Table
